## **Module Overview**

This comprehensive training module is designed to educate voters on how to participate effectively and securely in elections using the VoteAPI platform. It covers the complete voting process, security measures, and voter rights and responsibilities.

## **Learning Objectives**

By the end of this training, voters will be able to:
- Understand their rights and responsibilities as voters
- Navigate the voting system effectively
- Use secret codes securely and correctly
- Cast votes for all eligible positions
- Understand vote security and privacy measures
- Recognize and report voting issues
- Understand the election process and timeline

## **Section 3: Secret Code System**

### **3.1 Secret Code Overview**

#### **What is a Secret Code?**
- **6-Character Code**: Alphanumeric code (e.g., "ABC123")
- **Unique Code**: Each voter gets unique code
- **Non-Expiring**: Valid for entire election
- **Secure Delivery**: Sent via email and SMS
- **Vote Authentication**: Required for voting

#### **Code Purpose**
- **Identity Verification**: Verify voter identity
- **Access Control**: Control voting access
- **Security**: Prevent unauthorized voting
- **Audit Trail**: Track voting activity
- **Fraud Prevention**: Prevent vote fraud

### **3.2 Code Management**

#### **Code Delivery**
- **Email**: Sent to registered email address
- **SMS**: Sent to registered phone number
- **Timing**: Sent when registered by admin
- **Format**: Clear, readable format
- **Instructions**: Include usage instructions

#### **Code Security**
- **Keep Secret**: Don't share your code
- **Secure Storage**: Store code securely
- **No Screenshots**: Don't take screenshots
- **No Sharing**: Don't share with others
- **Report Loss**: Report if code is compromised

### **3.3 Code Usage**

#### **When to Use**
- **Voting Access**: Required to access voting
- **Each Position**: Use for each position vote
- **Entire Election**: Valid for entire election
- **Multiple Votes**: Use for all your votes
- **No Expiration**: Doesn't expire during election

#### **Usage Limits**
- **Maximum Attempts**: 3 attempts per voting session
- **Lockout**: 15-minute lockout after max attempts
- **Session Timeout**: 30-minute session timeout
- **Position Limit**: One vote per position
- **Candidate Limit**: One vote per candidate

## **Section 7: Election Timeline and Results**

## **Section 7: Election Timeline and Duration**

### **7.1 Understanding Election Duration**

#### **How Duration is Calculated**
- **Election timeline is calculated backwards** from the voting start date
- **All phases are scheduled** relative to when voting begins
- **Admin sets the voting start date** when creating the election
- **System automatically calculates** all other phase dates

#### **Example Timeline (Voting starts March 15th)**
```
March 1-7:     Election Setup (7 days before voting)
March 8-14:    Voter Registration (7 days before voting)
March 10-12:   Candidate Registration (3 days before voting)
March 13-14:   Campaign Period (2 days before voting)
March 15-17:   Voting Period (3 days)
March 18:      Results Processing (1 day)
March 19:      Results Announcement (immediate)
```

#### **Why This Timeline?**
- **7 days setup**: Enough time to configure election properly
- **7 days registration**: Ensures all voters are registered and codes delivered
- **3 days candidates**: Time for admin to review and add candidates
- **2 days campaign**: Fair campaign period for all candidates
- **3 days voting**: Ensures all voters can participate
- **1 day results**: Time for proper verification and certification

### **7.2 Voter Timeline**

#### **Your Timeline as a Voter**
1. **Registration Period** (7 days before voting)
   - Admin adds you to system
   - You receive secret code
   - You confirm registration

2. **Waiting Period** (6 days before voting)
   - Candidates are added by admin
   - Campaign period begins
   - You can review candidate information

3. **Voting Period** (3 days)
   - You can vote using your secret code
   - Vote for all eligible positions
   - Receive voting confirmation

4. **Results Period** (1 day after voting)
   - Results are processed and verified
   - Results are announced
   - You can view final results

- **Preliminary Results**: Real-time updates (every 30 minutes)
